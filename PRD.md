# PRD.md

# Hospital Data Migration Tool  
Product Requirement Document

---

# 1. Product Overview

본 프로젝트는 병원 데이터베이스 간 데이터 이관(Migration)을 지원하는 웹 기반 관리 도구입니다.

병원 시스템 교체 또는 데이터 통합 과정에서 발생하는 **대량 데이터 이관 작업**을 안전하게 수행하고  
사용자가 **이관 진행 상태를 시각적으로 확인할 수 있도록** 지원합니다.

본 시스템은 다음 기능을 제공합니다.

- 병원 데이터 이관 실행
- 이관 단계 진행 상태 표시
- API 기반 데이터 이관 처리
- 보안 강화를 위한 BFF 프록시 구조

---

# 2. Problem

병원 시스템 데이터 이관 과정에서는 다음과 같은 문제가 발생합니다.

## 2.1 이관 진행 상황 확인 어려움

기존 데이터 이관 작업은 백엔드 스크립트 기반으로 수행되는 경우가 많습니다.

문제

- 현재 진행 단계 확인 어려움
- 어느 단계에서 실패했는지 확인 어려움
- 사용자에게 진행 상태 전달 불가능

---

## 2.2 데이터 이관 순서 관리 문제

병원 데이터는 다음과 같은 **의존 관계**가 존재합니다.

예

```
환자 → 접수 → 오더 → 수납
```

순서를 잘못 처리하면 데이터 무결성이 깨질 수 있습니다.

---

## 2.3 보안 문제

DB 인증 정보를 프론트엔드에서 직접 처리할 경우 다음 위험이 존재합니다.

- DB 자격증명 노출
- 보안 취약점 발생
- 직접 DB 접근 가능

---

# 3. Goals

이 프로젝트의 주요 목표는 다음과 같습니다.

### 데이터 이관 실행 UI 제공

사용자가 데이터 이관 작업을 실행할 수 있어야 합니다.

---

### 진행 상태 시각화

데이터 이관 단계가 UI에서 표시되어야 합니다.

예

```
처방 → 환자 → 접수 → 오더 → 수납
```

---

### 안정적인 API 기반 이관

실제 데이터 이관은 API 서버에서 수행됩니다.

---

### 보안 강화

DB 인증 정보는 **프론트엔드에서 절대 노출되지 않습니다.**

이를 위해 **BFF Proxy 구조**를 사용합니다.

---

# 4. Target Users

본 시스템의 주요 사용자는 다음과 같습니다.

### 병원 IT 담당자

병원 시스템 데이터를 새로운 시스템으로 이관하는 작업을 수행합니다.

---

### 시스템 운영자

데이터 이관 진행 상황을 확인하고 오류를 대응합니다.

---

### 개발자 / 시스템 관리자

이관 API 상태를 확인하고 데이터 검증을 수행합니다.

---

# 5. Core Features

## 5.1 데이터 이관 실행

사용자는 병원 ID를 입력하여 데이터 이관을 시작할 수 있습니다.

입력 데이터

```
hospitalId
```

---

## 5.2 이관 진행 상태 표시

데이터 이관 과정은 UI에서 단계별로 표시됩니다.

총 단계

```
13 단계
```

단계 목록

```
1 처방
2 의약품·재료
3 용법 코드
4 템플릿 코드
5 묶음 항목
6 묶음 항목 상세
7 환자
8 접수
9 예약
10 상병
11 오더
12 활력징후
13 수납·정산·결제
```

---

## 5.3 UI 진행 시뮬레이션

API 응답을 기다리는 동안 사용자 경험을 위해 진행 애니메이션을 제공합니다.

특징

- 2.5초 간격 단계 진행
- API 완료 시 빠른 마무리

---

## 5.4 BFF Proxy 보안 구조

브라우저는 다음 데이터만 전송합니다.

```
hospitalId
```

프록시 서버가 다음 정보를 추가합니다.

```
source DB credentials
target DB credentials
```

프론트엔드에서는 **DB 인증 정보가 노출되지 않습니다.**

---

# 6. System Architecture

시스템 구조

```
Frontend (Vite + React)
        ↓
BFF Proxy Server (Express)
        ↓
Migration API
        ↓
Database
```

---

# 7. Non Goals

현재 MVP에서는 다음 기능을 제공하지 않습니다.

- 데이터 이관 로그 저장
- 실패 단계 자동 재시도
- 관리자 인증
- 다중 병원 이관 동시 처리

---

# 8. Success Metrics

프로젝트 성공 기준

### 데이터 이관 성공률

모든 이관 단계가 정상 완료

---

### 사용자 이해도

---

### 보안

DB 인증 정보가 프론트엔드에 노출되지 않아야 합니다.
