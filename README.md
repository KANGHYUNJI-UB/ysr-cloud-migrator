# 의사랑 클라우드 전환 헬퍼

병원 데이터를 **온프레미스(On-Premise) 데이터베이스에서 클라우드 데이터베이스로 이관**하는 내부 운영 도구입니다.

담당자가 **병원 ID를 입력하면 15단계 데이터 이관 작업이 자동으로 실행**되며,  
각 단계의 진행 상황이 **실시간으로 화면에 표시됩니다.**

이 도구는 병원 시스템의 클라우드 전환 과정에서 발생하는 **데이터 이관 작업을 안전하고 투명하게 관리하기 위해 설계되었습니다.**

---

# 문제 정의

병원 시스템을 클라우드로 전환할 때 다음과 같은 문제가 발생합니다.

### 데이터 이관 진행 상황 확인 어려움

기존 데이터 이관은 백엔드 스크립트나 배치 작업으로 수행되는 경우가 많습니다.

문제

- 현재 이관 진행 단계 확인 어려움
- 어느 단계에서 실패했는지 확인 어려움
- 운영자가 상태를 파악하기 어려움

---

### 데이터 의존 관계 문제

병원 데이터는 다음과 같은 **의존 관계**가 존재합니다.

```
환자 → 접수 → 오더 → 수납
```

이 순서를 잘못 처리하면 데이터 무결성이 깨질 수 있습니다.

---

### 보안 문제

데이터 이관 작업에서 DB 인증 정보가 프론트엔드에 노출될 경우 다음 위험이 있습니다.

- DB 자격증명 노출
- 직접 DB 접근 가능
- 보안 취약점 발생

---

# 해결 방법

이 프로젝트는 다음 방식으로 문제를 해결합니다.

- 데이터 이관 **단계별 진행 UI 제공**
- **실시간 진행 상태 표시**
- **BFF Proxy 구조로 DB 인증 보호**
- **SSE 기반 실시간 이벤트 스트리밍**

# 시스템 아키텍처

```
브라우저 (Vite :5173)
    │  POST /api/insert/all  { hospitalId }
    ▼
프록시 서버 (Express :3001)          ← DB 크리덴셜 보관
    │  DELETE /delete/all/:id
    │  POST   /stage/create
    │  POST   /prescription/insert
    │  POST   /patient/insert  ...    (15단계 순차 호출)
    ▼
이관 API 서버 (localhost:3000)
```

핵심 특징

- 브라우저는 `hospitalId`만 전송합니다.
- DB 접속 정보는 **프록시 서버에서만 관리됩니다.**
- 각 단계의 진행 상태는 **SSE(Server-Sent Events)**로 스트리밍됩니다.

---

# 이관 단계 (15단계)

데이터 이관은 다음 순서로 진행됩니다.

1. 기존 데이터 삭제  
2. 스테이징 생성  
3. 처방 정보  
4. 의약품·재료 정보  
5. 용법 코드 정보  
6. 템플릿 코드 정보  
7. 묶음 항목 정보  
8. 묶음 항목 상세 정보  
9. 환자 정보  
10. 접수 정보  
11. 예약 정보  
12. 상병 정보  
13. 오더 정보  
14. 활력징후 정보  
15. 수납·정산·결제 정보  

이 순서는 실제 데이터 의존 관계를 고려하여 설계되었습니다.

# 시작하기

## 사전 요구사항

- Node.js 18 이상
- 데이터 이관 API 서버 실행

```
localhost:3000
```

---

# 설치

```bash
git clone https://github.com/your-org/uisarang-cloud-migration.git
cd uisarang-cloud-migration
npm install
```

---

# 환경 설정

```bash
cp .env.example .env
```

`.env` 파일에서 DB 접속 정보를 설정합니다.

또는 다음 파일에서 직접 수정할 수 있습니다.

```
server/proxy.ts
```

설정 항목

```
DB_SOURCE
DB_TARGET
```

---

# 실행

```bash
npm run dev
```

브라우저 접속

```
http://localhost:5173
```
